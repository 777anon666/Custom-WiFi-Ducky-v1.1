<h3><a id="user-content-what-it-is" class="anchor" aria-hidden="true" href="#what-it-is"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>What it is</h3>
<p>It's a Wi-Fi controlled BadUSB device to remotely execute Ducky Scripts.</p>
<p>Using a USB device which act as a keyboard to inject keystrokes is well kown these days.
The <a href="https://hakshop.com/products/usb-rubber-ducky-deluxe" rel="nofollow">USB Rubber Ducky</a> by <a href="https://www.hak5.org/" rel="nofollow">Hak5</a> is THE hacker gadget for this kind of attack. It introduced a simple script language called <em>Ducky Script</em>, which this project uses too.</p>
<h3><a id="user-content-how-it-works" class="anchor" aria-hidden="true" href="#how-it-works"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How it works</h3>
<p>The ESP8266 is a popular Wi-Fi chip used in a lot of projects. Because it's cheap, small and has its own file system (<a href="https://github.com/esp8266/Arduino/blob/master/doc/filesystem.md">SPIFFS</a>), it's perfect for enabling an easy remote connection and holding Ducky Script payloads.</p>
<p>Although the ESP8266 is awesome, it doesn't have native USB, which means it can't act as a keyboard :(<br>
(<a href="https://github.com/cnlohr">cnlohr</a> made a cool project on this and added a USB stack himself: <a href="https://github.com/cnlohr/espusb">https://github.com/cnlohr/espusb</a>. The problem with that is, that it isn't compatible with the current SDK version, also I wanted to use Arduino to make it more user friendly.)</p>
<p>Here comes the ATmega32u4 into play!<br>
It can act as keyboard and thanks to <a href="http://youtube.com/seytonic" rel="nofollow">Seytonic</a> run Ducky Script (<a href="https://github.com/Seytonic/Duckduino-microSD">link</a>).<br>
So what I did is connecting the ATmega to the ESP8266 via serial.</p>
<p>The ESP will open up a Wi-Fi access point and host a webinterface from what you can upload and manage your scripts.<br>
When you hit run, it will send the script to the ATmega, which then will execute it on the target machine.</p>
<h3><a id="user-content-the-benefits-of-adding-wi-fi" class="anchor" aria-hidden="true" href="#the-benefits-of-adding-wi-fi"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>The benefits of adding Wi-Fi</h3>
<p><strong>But why add Wi-Fi</strong> ...you might ask.<br>
With Wi-Fi you can upload and run your Ducky Script payloads remotely.<br>
You just need to plug the device in, connect to its Wi-Fi network and you have full control over the target machine.</p>
<p>It also gives you one big advantage over other BadUSBs, you can test your scripts live! You don't need to copy them onto a micro-sd card or compile them. You can run them live over the webinterface, which makes its super easy for testing and improving your scripts.</p>
<p>It also adds a lot of possibilites for different attacks.<br>
You could make the target download executables from the Wi-Fi chip, instead of the internet.
Or execute different attacks and send the results back to the Chip. Or open up a reverse shell on the ESP8266s Wi-Fi.<br>
And so on... there are so much possibilities!</p>

You can either flash the bin file directly or compile it yourself using Arduino.</p>
<p><strong>Upload using Arduino</strong><br>
Open the <code>esp8266_wifi_duck</code> sketch with <a href="https://www.arduino.cc/en/Main/Software" rel="nofollow">Arduino</a>.
You need to install the following Librarys:</p>
<ul>
<li><a href="https://github.com/esp8266/Arduino">the latest ESP8266 SDK</a></li>
<li><a href="https://github.com/me-no-dev/ESPAsyncWebServer">ESPAsyncWebServer</a></li>
<li><a href="https://github.com/me-no-dev/ESPAsyncTCP">ESPAsyncTCP</a></li>
</ul>
<p>Then compile and upload it to your ESP8266 (check if your settings are right).</p>

<p>Open the <code>arduino_wifi_duck</code> sketch in Arduino and upload it to your ATmega32u4 "or" Arduino Based Board.</p>
<h3><a id="user-content-update-esp8266-over-the-webinterface" class="anchor" aria-hidden="true" href="#update-esp8266-over-the-webinterface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Update ESP8266 over the Webinterface</h3>
<p>Once you flashed the software, you can update it over the webinterface.<br>
Go to <code>192.168.4.1/update</code> and upload the new .bin file.<br>
(In Arduino go to <code>Sketch</code>-&gt;<code>Export compiled Binary</code> to compile your own .bin file)</p>

<p>Plug your Wi-Fi Ducky in and connect to the new Wi-Fi network <code>WiFi Duck</code>. The password is <code>quackquack</code>.<br>
Open your browser and go to <code>192.168.4.1</code>.</p>
